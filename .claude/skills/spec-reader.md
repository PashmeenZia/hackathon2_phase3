# Skill: spec-reader

## Purpose
Read and understand feature, API, database, and UI specifications.

## Description
This skill enables agents to parse, interpret, and extract actionable information from specification documents. It ensures that specifications are properly understood before implementation begins.

## Used By
- Main-Orchestrator
- Spec Manager
- Auth Agent

## Key Capabilities
- Parse specification documents (spec.md, plan.md, tasks.md)
- Extract requirements, constraints, and acceptance criteria
- Identify dependencies and relationships between components
- Validate specification completeness
- Surface ambiguities or gaps in specifications

## Usage Guidelines
- Always read the full specification before starting implementation
- Cross-reference related specifications (API, database, UI)
- Flag any unclear or contradictory requirements
- Ensure all acceptance criteria are understood
